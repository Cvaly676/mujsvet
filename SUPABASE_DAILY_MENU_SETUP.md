# Supabase + Vercel Daily Menu Setup

This project is prepared for:
- public website on Vercel
- daily menu data in Supabase
- simple admin page at `admin.html`

Use this order:
1) Supabase setup
2) local config test
3) Vercel deploy
4) domain connect
5) Supabase auth URL update

## 1) Create Supabase project

1. Open Supabase dashboard.
2. Create a new project.
3. Copy:
   - Project URL
   - anon public key

You will paste these in:
- `index.html` (`SUPABASE_URL`, `SUPABASE_ANON_KEY`)
- `admin.html` (`SUPABASE_URL`, `SUPABASE_ANON_KEY`)

## 2) Create `daily_menu` table

Run in Supabase SQL Editor:

```sql
create table if not exists public.daily_menu (
  id bigint generated by default as identity primary key,
  created_at timestamptz not null default now(),
  label_date text,
  week_start_date date,
  week_end_date date,
  name text not null,
  description text,
  price text,
  sort_order int not null default 100,
  is_active boolean not null default true
);
```

If table already exists, run:

```sql
alter table public.daily_menu add column if not exists week_start_date date;
alter table public.daily_menu add column if not exists week_end_date date;
```

Without these columns, save can still work in legacy mode, but week-range filtering will not be accurate until you add them.

## 3) Enable RLS + policies

```sql
alter table public.daily_menu enable row level security;

-- Public website reads only active rows
create policy "public_read_active_daily_menu"
on public.daily_menu
for select
to anon
using (is_active = true);

-- Logged admin can manage rows
create policy "auth_manage_daily_menu"
on public.daily_menu
for all
to authenticated
using (true)
with check (true);
```

If policy names already exist, drop/recreate.

## 4) Create admin account

1. Supabase -> Authentication -> Users.
2. Create user email/password.
3. This account logs into `admin.html`.

## 5) Fill keys in project files

In `index.html` and `admin.html` set:

```js
const SUPABASE_URL = 'https://YOUR_PROJECT.supabase.co';
const SUPABASE_ANON_KEY = 'YOUR_ANON_KEY';
```

Important:
- `anon key` is safe for frontend.
- never use service role key in site files.

## 6) Test locally first

1. Open `admin.html`
2. Login with admin user
3. Add a menu item
4. Open `index.html` and refresh
5. Confirm daily menu renders

## 7) Deploy to Vercel

1. Push repo to GitHub (if not already).
2. In Vercel: `Add New -> Project`.
3. Import this GitHub repo.
4. Framework preset: `Other` / static.
5. Deploy.

After deploy you get a URL like:
`https://mujsvet.vercel.app`

## 8) Connect your custom domain

In Vercel project:
1. `Settings -> Domains`
2. Add your domain (example `mujsvet.cz` and `www.mujsvet.cz`)
3. Vercel shows DNS records to add at your domain provider.
4. Add records exactly as shown.
5. Wait for DNS propagation.

## 9) Update Supabase auth URLs (critical)

Supabase -> Authentication -> URL Configuration:

- **Site URL**:
  - `https://yourdomain.tld`

- **Redirect URLs** (add all you use):
  - `https://yourdomain.tld/admin.html`
  - `https://www.yourdomain.tld/admin.html`
  - `https://your-project.vercel.app/admin.html`
  - local dev URL if needed

If this is wrong, login callbacks can fail.

## 10) Daily usage

- Go to `https://yourdomain.tld/admin.html`
- Login
- Option A: add/edit/hide/delete rows manually
- Option B (recommended): paste full daily menu text into bulk parser
  - choose `Platí od` and `Platí do` using date picker
  - click preview
  - save as structured menu
  - if parser fails, use fallback save as plain text
- Public page updates after refresh

## Quick troubleshooting

- Empty menu on homepage:
  - check `SUPABASE_URL` + `SUPABASE_ANON_KEY`
  - check `daily_menu` has rows with `is_active = true`
  - check public select policy exists
- Admin login fails:
  - verify user exists in Supabase Auth
  - verify Supabase URL config contains your domain/admin URL
- CORS/403 issues:
  - recheck project URL and key
  - verify deployed domain uses HTTPS
